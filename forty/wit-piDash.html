wit-piDash

Dynamic Content Dashboard

Introduction

This project has been in constant development for a few years now. It was the project that got me started with node.js. It's a project I continue to use to learn new technologies. It's a WOZiTech product, hence the "wit" prefix; short for "WOZiTech".

At its core is the Raspberry PI directly connected to a display device, typically a large TV; hence wit-pi. Content is served to the PI, which is then displayed; hence wit-piDash (Dash is short for Dashboard).

Showcasing this project is:

  • Raspberry PI - Model 2 or Model 3; running a minimal install of Raspbian with docker; this is wit-pi.
  • electron/VueJS - docker container running the "wit-Dash" application; presents as a desktop application
  • wit-piServe - docker container providing node.js docker application as the local web service to wit-Dash and connects to wit-serve, the WOZiTech central server to get and receive (push) content.
  • wit-piManager - docker container providing licensing of "wit-pi", "wit-piServe" and "wit-Dash", in addition to checking for and installing updates.

Links to this project include:

  • wit-serve - the primary server for all wit products providing licensing & authentication, in addition to being the main access point to the wit federated set of services; specifically here, for content.
  • wit-piAuth - running in a Docker container on wit-pi, allows for local authentication to access privilege content/services.
  • wit-piSign - running in a Docker container on wit-pi (Raspberry PI Model 3) interacts with content using gestures captured via web camera.
